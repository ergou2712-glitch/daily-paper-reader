## 摘要
模型卡片通常引用可信度基准测试分数，却未记录测量时间，且同一发布系列的连续检查点往往沿用相同的数值，仿佛其背后的模型并未发生变化。我们通过审计四个开源发布系列（Yi、Qwen、Mistral 和 Gemma）的各三个连续版本，在多种提示词模板下使用一组固定的可信度基准测试，验证了模型是否发生了偏移。结果显示，相邻版本间的平均绝对漂移显著高于基于独立性的无漂移参考零假设，且在移除基准测试、移除发布系列或切换到严格评分模式时，这种差距依然存在。因此，我们得出结论：归属于某一发布系列的可信度分数不应在未重新测量的情况下直接沿用到下一个检查点；相反，它应作为与特定检查点绑定且带有日期的产物进行报告，我们将其封装为“纵向模型卡片”。封闭 API、更大规模的模型、规范的基准测试协议以及固定的月度更新规则不在本次审计范围内，需要另行评估。

## Abstract
Model cards quote trust-benchmark scores without recording when they were measured, and the same number is routinely carried across successive checkpoints of one release line as if the model behind it had not shifted. We test whether it has shifted by auditing four open-source release lines, Yi, Qwen, Mistral, and Gemma, at three successive generations each, on a fixed basket of trust benchmarks under multiple prompt templates. Mean absolute adjacent-generation drift lands well above an independence-based no-drift reference null, and the gap persists when we drop a benchmark, drop a release line, or switch to strict scoring. We therefore conclude that a trust score attached to a release line should not be carried forward to the next checkpoint without remeasurement; it should instead be reported as a checkpoint-bound, dated artefact, which we package as a longitudinal model card. Closed APIs, larger models, canonical benchmark protocols, and fixed month-cadence rules lie outside the audited scope and require their own evaluation.

### 1. 核心问题与研究动机
*   **核心问题**：当一个模型系列（如 Qwen 或 Mistral）发布新版本（检查点）时，其模型卡片（Model Cards）通常会沿用旧版本的可信度基准测试分数。论文探讨了这些分数是否具有“可迁移性”，即旧的审计结果能否代表新检查点的表现。
*   **研究动机**：随着开源模型以月为单位快速迭代，预训练数据、对齐策略和模型规模都在发生变化。如果下游用户或监管机构盲目引用旧分数，可能会忽视模型在安全性、公平性或真实性上的潜在退化（漂移）。

### 2. 方法论
*   **核心思想**：通过对多个开源模型系列的连续代际进行固定基准测试，量化其可信度的波动，并将其与统计学上的“无漂移”基准进行对比。
*   **关键技术细节**：
    *   **评分漂移率（SDR）**：定义为相邻两代模型在同一基准测试下的得分差值（百分点，pp）。
    *   **汇总无漂移参考零假设（Pooled No-Drift Reference Null）**：利用二项分布和自助法（Bootstrap）模拟在没有真实性能变化、仅存在采样噪声情况下的预期漂移量，以此作为判断观测到的漂移是否显著的基准。
    *   **纵向模型卡片（LMC）**：提出一种扩展的模型报告模式，要求记录检查点哈希值、评估日期、所用提示词模板以及相对于前一版本的漂移值。

### 3. 实验设计
*   **审计对象**：4 个开源模型系列（Yi, Qwen, Mistral, Gemma），每个系列选取 3 个连续的公共版本（G1, G2, G3），共 12 个检查点。
*   **基准测试集（Benchmark Basket）**：
    *   **真实性**：TruthfulQA-MC1。
    *   **公平性**：BBQ-mixed, CrowS-Pairs-FC。
    *   **安全性**：ToxiGen-AT, XSTest-TD。
*   **对比维度**：
    *   使用了 3 种不同的提示词模板（T1, T2, T3）以排除提示词敏感性的干扰。
    *   对比了“宽松评分”（仅计算解析成功的样本）与“严格评分”（解析失败计为错误）。
    *   进行了“留一法”（Leave-one-out）扰动分析，分别移除特定基准或模型系列以验证结论的稳健性。

### 4. 资源与算力

*   **模型配置**：所有模型均在 4-bit NF4 量化下运行，使用贪婪解码（Greedy Decoding）。

### 5. 实验数量与充分性
*   **实验规模**：共进行了 180 组评估（12 检查点 × 5 基准 × 3 模板），涉及 36,000 次条目级决策。

### 6. 主要结论与发现
*   **显著的漂移**：平均绝对漂移量（|SDR|）为 **8.00 pp**，是无漂移参考值的 **3.6 倍**。
*   **性能退化普遍**：约 **47.5%** 的版本更迭出现了性能退化（Regression），尤其在真实性维度上，新版本得分往往低于旧版本。

### 7. 优点

*   **统计严谨**：引入了参考零假设（Reference Null）来区分“随机波动”与“系统性漂移”。
*   **实用性强**：提出的“纵向模型卡片”为模型治理和合规性披露提供了具体的、可操作的建议。

*   **混杂因素**：由于开源模型中间训练过程不透明，难以区分漂移是由预训练数据改变、对齐策略调整还是模型规模扩大引起的。
